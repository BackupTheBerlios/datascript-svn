Roadmap

  • We are defining a DataScript dialect supported by our implementation, documented in a DataScript Language Overview.
  • Relational DataScript will be a language extension that allows storing data in an SQL database, either in base type columns or in blobs. For the time being, we will support SQLite only.
  • We are working on an XML exchange format for DataScript modules and an XSL stylesheet that produces Javadoc style documentation.
  • A simpler and more robust parser and code generator has been developed based on antlr. The parser is complete, the code generator currently supports read() and write() methods for plain DataScript. Relational extensions are partially implemented and not yet functional. This reimplementation can be downloaded as rds-bin or rds-src package.
  • A C++ code generator will be developed.
